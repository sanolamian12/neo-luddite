# 260714 · 서버 배포 — git 체크아웃 복구 + 프로덕션 env 반영

## 요약

프로덕션 백엔드(Oracle 도쿄)에 `RAG_MIN_SCORE=0.50`을 반영하려다, **배포 구조 자체가 깨져 있던 것을 발견하고 고쳤다.** `/opt/neo-luddite`가 `.git` 없는 파일 복사본이라 `git pull` 배포가 애초에 불가능한 상태였다. `import-credigraph` 브랜치를 새로 clone해 교체하고 `.env`·`.venv`를 승계시켜, 이제 `git pull && systemctl restart`로 배포된다.

---

## 1. 발견 — 의도와 현실의 괴리

`backend/deploy/bootstrap.sh:39-44`는 분명히 git 방식이다.

```bash
REPO_URL="https://github.com/sanolamian12/neo-luddite.git"
BRANCH="import-credigraph"
APP_DIR="/opt/neo-luddite"
...
if [ ! -d "$APP_DIR/.git" ]; then
  git clone --branch "$BRANCH" "$REPO_URL" "$APP_DIR"
else
  git -C "$APP_DIR" pull
fi
```

그런데 실제 서버는 이랬다.

- `/opt/neo-luddite`에 `.git` 없음. `sudo find / -maxdepth 5 -name .git` → **시스템 전체에 git 체크아웃 0개**
- `/opt/neo-luddite` 아래에 `backend/`만 존재 (`frontend/` 없음) → clone이었다면 같이 받아졌을 것
- 실행 중인 프로세스 cwd = `/opt/neo-luddite/backend` (systemd `WorkingDirectory`)

즉 **의도는 git이었으나 실제로는 파일만 복사된 상태로 굳어 있었다.** 그래서 코드 배포를 하려면 매번 scp를 해야 했고, 서버 코드가 어느 커밋인지 추적할 방법이 없었다.

## 2. 접속 정보 (이번에 확정)

```bash
ssh -i docs/ssh-key-2026-07-09.key ubuntu@132.145.115.166
```

- 키는 **repo 안 `docs/`** 에 있다. git 추적은 안 됨(gitignore 적용 확인).
  다만 저장소 폴더 안에 private key가 있는 구조라 실수로 커밋될 여지는 남아 있음 — 정리 대상.
- `~/.ssh/id_ed25519`, `~/Documents/lifeline-*.pem` 은 이 서버 키가 아니다(전부 publickey 거부).

## 3. systemd 유닛의 함정 — env 변경 시 반드시 재시작

`neo-luddite-api.service`에 **`EnvironmentFile`이 없다.**

```
User=ubuntu
WorkingDirectory=/opt/neo-luddite/backend
ExecStart=/opt/neo-luddite/backend/.venv/bin/uvicorn api.main:app --host 127.0.0.1 --port 8787 --workers 1
```

앱이 `api/main.py:22`의 `load_dotenv()`로 `/opt/neo-luddite/backend/.env`를 **프로세스 시작 시 한 번** 읽는다.
→ `.env`만 고치고 재시작하지 않으면 아무 일도 일어나지 않는다.

## 4. 수행한 작업

### (a) RAG_MIN_SCORE 프로덕션 반영

```bash
cd /opt/neo-luddite/backend
cp .env .env.bak-$(date +%Y%m%d-%H%M%S)      # → .env.bak-20260714-065849
sed -i "s/^RAG_MIN_SCORE=.*/RAG_MIN_SCORE=0.50/" .env
sudo systemctl restart neo-luddite-api
```

`.env` 17번 줄이 `0.0` → `0.50`. (산정 근거는 RAG 개선 문서 참조)

### (b) 파일복사본 → git 체크아웃 교체

```bash
# 1) 임시 위치에 clone
sudo git clone --branch import-credigraph https://github.com/sanolamian12/neo-luddite.git /opt/neo-luddite-new

# 2) .env / .venv 승계 (venv는 절대경로라 위치만 같으면 재설치 불필요)
sudo cp -a /opt/neo-luddite/backend/.env  /opt/neo-luddite-new/backend/.env
sudo mv    /opt/neo-luddite/backend/.venv /opt/neo-luddite-new/backend/.venv
sudo chown -R ubuntu:ubuntu /opt/neo-luddite-new

# 3) 교체 (구본은 롤백용 보존)
sudo systemctl stop neo-luddite-api
sudo mv /opt/neo-luddite     /opt/neo-luddite-filecopy-backup
sudo mv /opt/neo-luddite-new /opt/neo-luddite
sudo systemctl start neo-luddite-api
```

디스크 여유는 충분했다 (`/dev/sda1 45G, 6.3G 사용, 39G 여유`).
