## 문제 (사용자 제보, 순차)

- 일감 상세에서 **포함된 대화가 "—", "0 turns"** 로만 뜸 → 라이브 대화(`live…`)가 정적 번들에 없어 lookup 실패.
- 3-패널(진행중 큐 / 본문 / 검수)이 **서로 스크롤 안 됨** → 큐 패널의 피드백 카운트가 모니터에 따라 안 보임. 큐 가로 리사이즈 + 우측 패널도 접기 요구.
- 큐 도트 색 의미가 모호 → **회색=내코멘트0 / 주황=내코멘트≥1** 로 확정(제출 여부는 별개 축). 배지 숫자는 **총 코멘트**.
- 파란 말풍선 안 텍스트를 드래그하면 **흰 글자가 사라져 보임** → 선택 하이라이트가 밝아서. 이어서 "말풍선 포커스 시 전체가 흰색"이 근본 원인.
- 목록의 **[시작][취소] 버튼 정렬 깨짐**(셀 밖 클리핑) + 클릭 시 **"대화 데이터를 찾을 수 없습니다" 종종 표출**. 취소는 상세로 옮겨 **[참여 철회]** 로.
- 진행중 패널에서 **[시작전]과 [작성중] 목록이 동일**하게 보이고, **제출됨이 초록이 아닌 노랑**으로 표시됨.

## 큐 도트/필터 모델 (최종 확정 — 이번 세션에 한 번 뒤집혔다 복귀)

`56fb99d`에서 사용자가 "여러 상태 혼동"이라 해 **2-상태(회색/주황, 내 코멘트만)** 로 단순화했으나, 실사용 후 "제출됨은 초록이어야" + "시작전·작성중 목록이 같다" 재요청 → `b65f1db`에서 **3-상태로 복귀**. 최종:

| 상태 | 도트 | 조건 | 필터 |
|---|---|---|---|
| 시작전 | 회색 `bg-muted-foreground/40` | `draft && 내 코멘트 0` | [시작전] |
| 작성중 | 주황 `bg-amber-500` | `draft && 내 코멘트 ≥1` | [작성중] |
| 제출됨 | 초록 `bg-emerald-500` | `isSubmitted`(submitted/reviewed/finalized) | [제출됨] |

- 제목 옆 숫자 배지 = **이 대화의 총 코멘트**(작성자 불문). 도트·필터 = **내 코멘트/내 제출**. 두 축을 분리한 게 핵심.
- 도트와 필터가 **같은 판정(`dotClass`/`isSubmitted`)을 공유** → 색과 목록이 항상 일치.

## 남은 것 / 후속

- `cancelAudit`는 내 기여 기준으로 고쳤으나, **`audit.progress.feedbackCount` 자체는 여전히 총합**(audit-workspace 동기화가 conversationId만으로 집계). 표시/커버리지 등 다른 소비처에 남아 있음 — 필요 시 progress 의미를 '내 기여'로 일원화 검토(이번엔 리플 우려로 미변경).
- 큐 리사이즈 폭·검수 패널 접힘 상태는 **미영속**(새로고침 시 기본값). 필요하면 localStorage 영속.
- history 폴더에 다른 세션의 untracked `260711_세션총평_…설계방향.md`가 별도로 있음(내 커밋에 미포함).
