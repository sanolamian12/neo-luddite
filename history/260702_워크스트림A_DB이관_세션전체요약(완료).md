# 워크스트림 A — DB 이관 세션 전체 요약 (Supabase 도입 → 서비스 컷오버)

날짜: 2026-07-02 · 브랜치: `import-credigraph` · 지휘문서: [260702_마스터설계_ABC_워크스트림_분리실행.md](260702_마스터설계_ABC_워크스트림_분리실행.md)

## 한 줄 요약
워크스트림 A(Seam B 플랫폼 상태의 Supabase 이관)를 **코드 완성 → 라이브 프로젝트 적용 → 프론트 서비스/스토어 전면 컷오버 + Realtime 동기화**까지 한 세션에서 관통. `tsc` 통과·스키마/RLS/Realtime 정적 대조 완료. 남은 건 두 브라우저 로그인 런타임 검증(완료정의).

## 이 세션의 작업 순서 (전 과정)
1. **A 코드 계층 커밋 정합화** — 루트 `.gitignore` 가 `supabase/` 와 `frontend/.env.example` 을 통째로 무시하던 문제 발견·수정(`!/supabase/`, `!.env.example`). A DB 계층이 커밋에서 누락될 뻔한 것을 방지. → 첫 커밋.
2. **Supabase 라이브 프로젝트** — 대시보드에서 `credigraph` 생성(리전 **Tokyo ap-northeast-1**; Seoul 생성 제한으로 대체, ~30-40ms 허용). URL/anon/service_role 확보 → `frontend/.env.local`·`backend/.env` 채움(gitignore, 미커밋).
3. **스키마·시드 적용** — Supabase CLI 를 스크래치에 격리 설치 → 세션 풀러로 `db push`(0001·0002 이력 기록) + `seed.sql`(pg 로 직접). 검증: public 15테이블 전부 RLS · Realtime 5테이블 · 트리거 profiles 3 자동생성 · 역할/도메인 매핑 정확.
4. **서비스/스토어 전면 컷오버** — 서비스 10 + 스토어 10 을 Zustand+localStorage → Supabase 원천 + Realtime 캐시로 교체. 공용 헬퍼 `lib/supabase/sync.ts`. UI 무손상(직접 mutation 1곳만 교체). review.finalize 교차도메인 버그 수정. 마이그레이션 0003 으로 Realtime publication 을 11테이블 전부로 확장.

## 핵심 설계 결정
- **시그니처 불변(§3-3)**: text PK 보존 · epoch-ms bigint(`Number()` 왕복) · jsonb=zod 오브젝트 1:1(camelCase 보존) · `profiles.domain_id` 로 `auth.uid()`↔도메인 텍스트 id 임피던스 흡수.
- **왜 "서비스만 교체"로 부족했나**: 컴포넌트가 스토어를 반응형 구독 → 스토어를 DB fetch+Realtime 캐시로 만드는 레이어가 필수(이전 세션 문서가 라이브 없이 못 본 부분).
- **읽기-수정-쓰기**(pickups·messages·decisions·balance 등 jsonb/배열)는 최신 행을 DB fresh fetch 후 수정.
