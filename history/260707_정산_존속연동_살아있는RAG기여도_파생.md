# 정산 존속연동 — 분배기준을 ledger → 살아있는 RAG 기여도로 교체

날짜: 2026-07-07 · 브랜치: `import-credigraph` · 선행: [260707_검수_두게이트_저장_최종승인_retract부채소멸.md](260707_검수_두게이트_저장_최종승인_retract부채소멸.md) · [260704_정산_설계_핸드오프.md](260704_정산_설계_핸드오프.md)

## 한 줄 요약
`settlement.preview()` 의 분배 기준을 **기간 내 ledger 인정건수** → **지금 RAG 에 살아있는 기여도**(`rag.passages status='active'` 의 auditor_id 별 count)로 교체했다. 포장실에서 [연결 끊기](retract→status=retired)하면 그 passage 가 집계에서 빠져 해당 세무사 기여도가 **자동 감소**한다 → "버려지면 기여도 소멸"이 별도 저장이 아니라 **조회의 파생**으로 성립. 라이브 DB 로 파생 감소를 실증(auditor 2→1→복원, DB pristine).

## 선결조건 (이미 갖춰져 있었음)
- passage 에 `auditor_id` attribution 실림(0705 공용검수보드).
- "확정=적재" 시점이 **최종승인 단일 게이트**로 명확(0707 두 게이트). → 남은 건 파생 로직 하나뿐.

### 프론트 (분배 기준 교체 — preview 하나에 격리)

- **[components/admin/settlement-new-form.tsx](../frontend/components/admin/settlement-new-form.tsx)** — 라벨: "인정 건수 비례"→"기여도 비례", "인정 피드백 합계"→"활성 기여 합계", 배지 "인정 N"→"기여 N", audit건수 span 제거, 기간 힌트/빈 미리보기 문구 갱신.
- **[components/admin/settlement-detail-view.tsx](../frontend/components/admin/settlement-detail-view.tsx)** — MODEL_LABEL·요약·"인정"→"기여" 컬럼. "포함 audit" 셀은 값 없으면(신규 RAG 회차) `—`.

## 설계 결정
- **기간(created_at) 옵션 필터**를 SQL 에 얹음(핸드오프 예시 SQL 의 superset). 폼의 기간 입력이 계속 의미를 갖고, "그 기간에 적재됐고 **지금도 살아있는**" 기여만 집계 = "기여=RAG 존속기간"을 회차 기간 안에서 성립. 회차 기간이 안 겹치면 중복지급도 자연 방지 → **구 settle-once dedup 대체**(그래서 제거 가능).
- 분배 **모델(even/weighted)** 형태는 유지, **기준만** 교체. 이미 발행된 회차의 저장 allocations(acceptedCount·includedAuditIds)는 그대로 렌더(하위호환).
- preview 가 이제 **네트워크(Seam A) 의존** — 백엔드 미기동이면 빈 미리보기. (구 preview 는 순수 프론트 ledger 였음.)

- **미실행**: 브라우저 육안(정산 만들기→발행→세무사 화면). 백엔드 기동 필수(`NEXT_PUBLIC_API_BASE`).
