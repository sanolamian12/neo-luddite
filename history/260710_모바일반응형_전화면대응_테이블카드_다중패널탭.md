# 모바일(412px) 반응형 — 전 화면 대응

날짜: 2026-07-10 · 브랜치: `import-credigraph` · 커밋: `d638888` (49 files, +2343/−811) · 목적: 라이브(neo-luddite.vercel.app)를 휴대폰에서 열면 화면이 짤려 버튼·리스트가 안 보이던 문제 해결. 기준 뷰포트 = Galaxy S20 Ultra **412×915**.

## 문제 (사용자 제보)

휴대폰에서 여러 화면이 짤림 → 버튼이 안 보여 클릭 불가, 리스트 항목이 안 보임. PC 모니터에선 공간이 넓어 문제없던 것들. **모바일 전용**으로 뷰 문제만 해결(데스크톱 레이아웃 불변).

요구 3가지: ① 여러 컴포넌트 동시표시가 안 되면 **탭**으로 ② 카드/리스트에서 긴 ID는 **앞4…뒤4** 축약하고 버튼은 꼭 다 보이게 ③ 한 화면에 담기 어려운 컴포넌트는 **press→팝업 다이얼로그**로.

## 접근 원칙

- **`useIsMobile` 훅 안 씀** → 순수 Tailwind `md:`(768px) 브레이크포인트 클래스로만 분기. 하이드레이션 깜빡임 방지, 데스크톱 원상 보존.
- 넓은 테이블 처리 방식은 사용자와 확정 → **카드 전환 통일**(가로스크롤/하이브리드 아님). 일관성·버튼 항상 노출.
- UI 라이브러리는 **base-ui**(`@base-ui/react`, Radix 아님). Tabs·Dialog는 없어서 신규 래퍼 작성. **주의: base-ui Tab 활성 상태 = `data-active`**(not `data-selected`).

## 공용 primitive (신규)

- `lib/utils.ts` `middleTruncate(s, head=4, tail=4)` — "앞4…뒤4" 축약. 항상 `<span title={원문}>{middleTruncate(id)}</span>`(원문은 title 툴팁).
- `components/ui/tabs.tsx`, `components/ui/dialog.tsx` — shadcn 스타일 base-ui 래퍼. (Sheet는 기존에 있었음)

## 3대 패턴 (적용 내역)

1. **넓은 테이블 → 모바일 카드**: `<div className="hidden overflow-x-auto md:block"><table>…</table></div>` + `<ul className="divide-y md:hidden">`(카드: 제목+축약ID / 상태배지 / `dl grid-cols-2` 지표 / `flex flex-wrap` 버튼전부). 행 계산 로직은 desktop/mobile map이 공유. 기존 `overflow-hidden`(← 우측 열 클리핑 원흉) 제거.
   - 적용: auditors·inspection·settlement·tasks·version-list·batch-list·pool·mail·packaging-list(admin), work·results·ledger(2표)·(queue-table은 이미 카드)(auditor).

2. **다중 패널 워크스페이스 → 모바일 탭**: 부모에 `mobileTab` state + `md:hidden` 탭바, 자식 pane에 `mobileShow?: boolean` prop 추가 → root class `cn("… md:flex", mobileShow ? "flex" : "hidden md:flex")`.
   - **inspection-workspace**(전사/결정): 결정 패널이 원래 `aside hidden md:flex`라 **모바일서 인정/거절 UI 자체에 접근 불가**였음(기능 소실) → 최우선 복구. 전사 피드백 칩 탭하면 자동 결정탭 전환.
   - **audit-experience · work/audit-workspace**(큐/전사/검수 3탭): 좌 큐스트립·우 인스펙터가 `hidden md:flex`였던 것 동일 처리.
   - topbar: 내보내기 버튼 라벨 모바일 아이콘화, audit/task ID 축약.

3. **고정 320px aside 2패널 → 모바일 목록↔상세 전환**: `mobileView: "list"|"detail"` state, 항목 탭하면 상세, 상단 `md:hidden` "← 목록" 뒤로가기.
   - **mailbox-view**(auditor), **inquiries-view**(admin).

4. **press→Dialog** (요구 ③): **포장실 packaging-detail passage** — UUID+dedupeKey+본문+태그가 한 카드에 과밀 → 모바일 전용 "상세" 버튼(`md:hidden`) press 시 전체 정보 Dialog. (분석 결과 대부분의 과밀은 탭·카드로 해결됐고 이게 남은 진짜 후보)

부수: 긴 ID 원문 노출 전 지점 축약, 반응형 없던 `grid-cols-4 divide-x`→모바일 2×2, `flex-wrap` 없던 버튼줄(pool-table·task-create·pool-detail·pipeline 등) 정리, kb 경로/인라인코드 `break-all`·`overflow-x-auto`.

## 실행 방식

- 카탈로그: Explore 에이전트 3개(admin/audit/chat·auditor·layout)로 412px 문제 지점 전수 조사.
- 기계적 대량 변경(테이블→카드, ID축약, grid·버튼): general-purpose 에이전트 4개 병렬(파일셋 disjoint, 정확한 카드 패턴 스펙 전달). 복잡한 상태 리프팅(워크스페이스·2패널 탭)은 직접.
