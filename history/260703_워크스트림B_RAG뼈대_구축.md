# 워크스트림 B — pgvector RAG 뼈대 구축 (빈 KB 출발형)

날짜: 2026-07-03 · 브랜치: `import-credigraph` · 지휘문서: [260702_마스터설계_ABC_워크스트림_분리실행.md](260702_마스터설계_ABC_워크스트림_분리실행.md)

## 한 줄 요약
RAG 를 "판례 전량 인덱싱"이 아니라 **세무사 코멘트(질문 A+답변 B+코멘트 C)로 자라는 KB** 로 재정의(임팩트 증명이 목표라 **빈 KB 로 출발**)하고, 그 성장 루프의 뼈대 — `rag.*` 스키마·Retriever 경계·Upstage 임베딩·write path(ingest)·pipeline③ 스텁 교체·RAG on/off 임팩트 스위치 — 를 한 세션에서 구축·오프라인 검증. 라이브 적용(DB URL·마이그레이션 push·시드)만 잔여.

## 제품 논지 재정의 (이 세션의 핵심 결정)
메모리 [[project_rag_product_thesis]] 신설. 질문 A + Upstage 답변 B + **인간 세무사 코멘트 C** → 이 C 가 RAG source. "이렇게 사람이 키운 RAG 가 효과가 있는가/얼마나"를 보이는 게 대회 임팩트.
- **결정적 함의**: `backend/data/.../all_cases.jsonl`(1301건) **전량 사전 인덱싱 금지** → before/after 대비가 사라짐. 최소 기본 지식만 시드.
- 실제 KB 는 배포(C) 후 세무사 ~100샘플 검수 코멘트가 채움 = 하차장→검수→포장실→RAG 루프의 write path.
- 실행 순서 확정: **① B 뼈대 → ② Seam A 프론트 배선(use-remote-runtime) → ③ C 배포 병행.**

## 이번 세션 산출물
**DB (마이그레이션 0004):**
- `supabase/migrations/0004_rag_schema.sql` — `rag` 스키마 + pgvector + `rag.passages`.
  - passage = Q+A+C 번들 텍스트(`content`) + `embedding vector(4096)` + **provenance 1급 컬럼**(reviewer·conversation_id·segment_id·feedback_id·kb_document_id·case_id·tax_category·occupation·case_refs·law_articles·feedback_tags).
  - `dedupe_key unique` 로 멱등 업서트. 방어적 RLS(정책 없음 = service role 만 통과).
  - `rag.match_passages(query_embedding, k, occupation?, tax_category?)` 코사인 top-k 함수.
  - **임베딩 차원 4096 실측**(embedding-passage/query 둘 다). pgvector ANN 인덱스는 2000차원 상한이라 4096 은 인덱스 불가 → **정확 코사인 검색**(빈 KB 출발이라 충분).

**백엔드 (`backend/api/rag/`):**
- `embeddings.py` — Upstage `embed_passage`/`embed_query`(4096d, 차원 검증). `api.llm.get_client()` 재사용.
- `store.py` — psycopg3+pgvector 로 Supabase 직결. `search()`·`upsert(PassageRecord)`·`count()`·`is_configured()`.
- `retriever.py` — `Retriever` 경계(§3-3) + `Passage` + `get_retriever()` 팩토리. **graceful**: DB 미설정/장애/빈 KB → `NullRetriever`(빈 결과, 챗 정상 = baseline). `rag_enabled()` 스위치.
- `ingest.py` — write path. `build_bundle_text(Q,A,C,tags)` + `ingest_feedback`/`ingest_kb_document`/`ingest_case_seed`.
- `scripts/rag_ingest.py` — CLI: `seed --n`(기본 지식 시드)·`count`·`search "질의"`(스모크).

**파이프라인 배선:**
- `pipeline.py` ③단계: `_CASE_REF` 정규식 스텁 → `get_retriever().retrieve()` 로 교체(정규식은 보조 병합만). RAG 지식을 `write_segments` 그라운딩에 주입(판정은 여전히 엔진 권위 — 뒤집지 않음). `rag_override` 파라미터.
- `llm.write_segments(..., rag_passages=)` — 참고 지식 블록 추가.
- `schema.py` `ChatMeta.ragHits` 추가(임팩트 측정).
- `main.py` — `POST /api/chat?rag=false` 요청 단위 A/B 오버라이드 + `GET /rag/health`(KB 크기·설정 점검).

**설정:** `.env.example`·`requirements-api.txt`(psycopg[binary]·pgvector) 갱신. 신규 env: `SUPABASE_DB_URL`·`RAG_ENABLED`·`RAG_TOP_K`·`RAG_MIN_SCORE`.
