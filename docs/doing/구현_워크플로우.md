```mermaid

    IN[세무 문서<br/>PDF · 스캔] --> C1["① 비식별화 전처리기<br/>주민/사업자번호 · 납세내역 마스킹<br/>(파이프라인 첫 노드)"]
    C1 --> C2["② 문서 파이프라인<br/>Document Parse → Information Extract<br/>📡 Upstage API"]
    C2 --> C3["③ Expert Answer Pool<br/>케이스 1건 × N명 독립 초안<br/>마감 관리 · 답안 버저닝"]
    C3 --> C4["④ Peer Review Graph (Neo4j)<br/>N-1 순환 검수 · diff = directed edge<br/>1건당 N² 감독 신호"]
    C4 --> C5["⑤ Consensus Engine<br/>자동 판정 → 관리자 검토 → 감사 로그<br/>📡 Solar Pro API"]

    KA -. "도메인 스키마 튜닝" .-> C2

    CR --> OUT1((합의 데이터셋))
```

## 핵심 읽는 법

- **워크플로우(1)**: LLM 호출은 ②·⑤ 단 2곳, 나머지 4개 컴포넌트와 KA→프롬프트/RAG 피드백 루프가 "시스템 러닝"의 실체.
- **간트(2)**: A의 *전문가 10명*(6/15)과 B의 *케이스 100건*(6/15) 두 사람의 산출물이 6월 중순에 동시 도착해야 C의 ④ Peer Review Graph가 빈손이 아닌 채로 시작 가능. 여기서 한 주만 밀려도 7/6 파일럿 시작이 깨짐.
- **핸드오프(3)**: A의 *자문 결론*은 C의 ⑤·KPI 둘 다에 입력. B의 *비식별화 정책*은 ①의 코드 직전제. B의 *5/21 워크숍 답변*은 트랙 컴플라이언스 가드레일로 ②·⑤에 반영.
