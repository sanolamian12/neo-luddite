> 팀: 네오러다이트 · 프로젝트: CrediGraph · 작성: 2026-05-23
> 비개발자 팀원도 핵심을 이해할 수 있게 정리. 근거·기술 상세·코드 가드레일은 [기술스택·인프라 결정사항 원본](260523_기술스택_인프라_결정.md) 참조.

## 1. 한 줄 요약

CrediGraph 는 세무 문서가 **6단계 파이프라인**을 통과하면서 합의 데이터·전문가 기여 기록·AI 학습용 컨텍스트로 누적되는 시스템. AI는 **Upstage 제품군**만 사용하고, 우리가 짜는 코드(약 80%)는 그 외부 AI를 둘러싼 워크플로우·그래프·원장·웹화면.

---

## 2. 6개 컴포넌트 이름표

문서·회의에서 "컴포넌트 ①" 같은 번호가 나오면 다음을 가리킴:

| # | 이름 | 하는 일 |
|---|---|---|
| ① | **비식별화 전처리기** | 세무 문서의 개인정보(주민번호·이름·주소 등)를 가린다 |
| ② | **문서 파이프라인** | PDF/스캔 → 정형 JSON 데이터로 변환 (Upstage Document Parse·Information Extract 호출) |
| ③ | **Expert Answer Pool** | 케이스 1건에 전문가 N명이 독립적으로 1차 답안 작성 |
| ④ | **Peer Review Graph** | 전문가들이 서로의 답안을 검수, 수정 이력을 그래프로 저장 |
| ⑤ | **Consensus Engine** | Upstage Solar Pro 3 가 합의 후보 생성 → 관리자 검토 → 감사 로그 |
| ⑥ | **Contribution Ledger + Credit Scoring** | 누가 무엇을 기여했는지 변조 불가능한 원장에 기록 + 크레딧 정산 |

→ 이 6개를 묶으면 "전문가들이 답하고 서로 검수하는 과정 자체가 자동으로 학습 데이터·지식 자산·정산 근거로 누적되는 시스템". 자세한 설계 배경은 [260516 프로덕트 설계](../260516%20프로덕트%20설계.md).

## 3. AI 모델 ─ Upstage 단일 사용 ⚠️ 가장 중요

본 대회는 **국내 AI 트랙**이라 외산 AI를 한 번이라도 호출하면 일반 트랙으로 강등됩니다. 우리는 **Upstage 사 제품만 단독 사용**으로 확정 (2026-05-17 결정).

| Upstage 제품 | 우리 시스템에서 쓰는 곳 |
|---|---|
| **Solar Pro 3** | 합의 판정 (⑤), 사용자 채팅 응답 |
| **Document Parse** | PDF → 정형 데이터 (②) |
| **Information Extract** | 필수 필드 추출 (②), 이름·주소 같은 개인정보 추출 (①) |
| **Solar Embedding** | "의미가 비슷한 케이스" 검색용 (⑤의 RAG) |

**팀 전원 금지 사항** — 학습·합성 데이터 생성·검수·평가 어느 단계에서도:
- ❌ OpenAI(GPT) / Anthropic(Claude) / Google(Gemini) / xAI(Grok) API
- ❌ Llama / Mistral / Qwen / Gemma / Phi / DeepSeek 직접 추론
- ❌ 외산 임베딩(BGE, E5 등), 외산 리랭커, 외산 모델로 채점(LLM-as-judge)

판정 기준은 "어떤 라이브러리를 썼나"가 아니라 **"어떤 모델 가중치를 추론에 썼나"**. 자세한 가이드: [HF_사용정책_국내AI트랙.md](HF_사용정책_국내AI트랙.md).

## 4. 백엔드 (서버 측) 기술

| 무엇 | 도구 | 비개발자가 알아둘 점 |
|---|---|---|
| 프로그래밍 언어 | Python 3.11 | 제안서 명시 |

| 로깅 | structlog | KPI 측정에 필요한 모든 이벤트를 표준 형식으로 기록 |

## 5. 데이터 저장 ─ 4종 데이터베이스 + 파일 저장소

| 저장소 | 담는 내용 |
|---|---|
| **PostgreSQL** | 케이스, 답안, 검수, 사용자, RBAC, **크레딧 원장(⑥)** 등 기본 데이터 |
| **pgvector** | "의미가 비슷한 문장" 검색용. PostgreSQL의 확장 기능 — 별도 DB 아님 |
| **Neo4j** | 전문가↔답안↔수정이력의 관계 그래프 (**Peer Review Graph ④**) |
| **Redis** | 작업 대기열, 임시 캐시 (AI 응답 24시간 캐싱) |
| **S3** (대회 제공) | 원본 PDF·스캔 파일, 비식별화된 사본 |

### "벡터 DB(pgvector)"가 왜 필요한가

일반 DB는 *정확히 일치하는 행*만 찾지만, 벡터 DB는 *의미가 가까운 행*을 찾음. 예: 새 케이스가 "병원 진료비 공제 문의" 일 때, 과거 합의 케이스 중 "의료비 공제 한도" 류를 자동으로 끌어와 AI 답변의 근거 자료로 넣어줌. 이게 AI가 점점 똑똑해 보이는 핵심 장치.

## 6. 사용자 화면 ─ 한 웹사이트, 4 종류

같은 도메인 안에서 **로그인한 사람의 역할에 따라 다른 화면**이 보임:

| 경로 | 누가 보는가 | 용도 | 본선 산출물 매핑 |
|---|---|---|---|
| `/` | 일반 사용자 (세무 의뢰인·소상공인) | ChatGPT 스타일 채팅 — 세무 질문, 문서 업로드 | (본선 시연 핵심) |
| `/expert` | 세무 전문가 10명 | 1차 답안 작성 / 다른 전문가 답안 검수 | 합의 데이터셋 생산 |
| `/admin` | 관리자 (서비스 운영) | 합의 검토, 크레딧 정산, 이의제기 처리 | **② 기여 대시보드** |
| `/api` | 외부 호출자 (심사·도입처) | 합의 엔진 호출 API | **③ 합의 엔진 REST API** |

### 화면 기술 결정

- **Next.js + TailwindCSS** (모던 웹 표준 조합)
- **노트북·데스크톱 메인** + 모바일 반응형 (앱은 본선 후 결정 — 세무 문서 업로드는 노트북이 주력 시나리오)
- 채팅 UI는 오픈소스 챗봇 베이스(`assistant-ui` 등)에 백엔드만 Upstage로 교체 → 직접 구현 시간 절감
- 다른 전문가의 답안 수정 이력은 **diff(빨강·초록 줄)** 와 **그래프 시각화**로 보여줌 (혁신성 심사 직격 요소)

## 7. 개인정보 보호 (PII)

세무 문서엔 민감 정보가 많아 **두 단계**로 가림:

1. **정규식 1차** — 주민/사업자/계좌/카드번호처럼 형식이 정해진 건 자동 식별·마스킹
2. **Upstage Information Extract 2차** — 이름·주소·전화번호처럼 문맥이 필요한 건 AI가 찾아서 마스킹

→ 외부 한국어 개체명 인식 모델(KoBERT 류)은 트랙 회색지대라 **미사용**. Information Extract 가 그 기능을 대체.

## 8. 보안 ─ 위변조 방지 (크레딧 정산의 신뢰성)

전문가 크레딧이 공정하려면 기록이 변조 불가능해야 함:

- **SHA-256 해시 체인** — 모든 기록이 직전 기록의 지문을 참조해 사슬처럼 묶임. 한 줄만 바뀌어도 사슬이 깨져 즉시 탐지
- **수정·삭제 차단** — 원장 테이블은 데이터베이스 차원에서 추가만 가능 (UPDATE/DELETE 금지)
- **매일 자동 무결성 검증** — 백그라운드 작업이 매일 사슬을 처음부터 끝까지 검증
- 정정이 필요하면 기존 기록을 *고치지 않고* **차액 기록을 새로 추가**

→ 본선 데모에서 "위변조 시도 시 사슬이 깨지는 모습"을 라이브 시연 가능 — 안전성 심사 직격 ([260516 프로덕트 설계 §5](../260516%20프로덕트%20설계.md)).

## 9. 사용자 권한 ─ 4 가지 역할

자체 RBAC(역할 기반 접근 통제) 시스템. A 팀원이 전문가 계약 시 부여 역할 확인 필요:

| 역할 | 권한 |
|---|---|
| `admin` | 모든 화면, 합의 최종 확정, 크레딧 조정, 이의제기 결정 |
| `expert` | `/expert` 답안 작성·검수, 자기 크레딧 조회 |
| `reviewer` | `expert` + 다른 전문가의 답안 다중 검수 권한 |
| `end_user` | `/` 채팅 사용, 자기 의뢰 케이스 조회 |

## 10. 우리 팀이 알고 있어야 할 것 / 행동 사항

| 누구 | 무엇 |
|---|---|
| **전원** | 외산 AI 모델(GPT·Claude·Gemini·Llama 등) **어떤 단계에서도** 사용 금지. 데이터 가공·평가·합성 모두 포함 |
| 팀장 | 본선 발표 시 강조할 차별점: ① Upstage 단독 사용 (국내 AI 트랙 100% 충족), ② SHA-256 위변조 방지 원장, ③ 전문가 합의 그래프 |
| A · 도메인/전문가 | 전문가 계약 시 부여 역할은 `expert` 또는 `reviewer` 중 하나. NDA에 데이터 접근 범위 명시 필요 |
| B · 데이터/운영 | 케이스 100건의 원본 PDF 는 S3에, 비식별화 사본은 S3 별도 폴더에 보관. **외산 LLM API로 데이터 가공·평가 절대 금지** |
| C · 개발 | CI에 외산 모델 ID grep 룰(`gpt-`, `claude-`, `meta-llama/` 등) 박기 — 발견 시 빌드 실패 |
