**2\. 기술 명세**

| 2.1 기술목적 | AI 고도화로 전문직의 반복 업무는 빠르게 자동화되고 있지만, 예외 판단, 규제 해석, 최종 검토, 상호 검수와 같은 고부가 개입은 여전히 사람의 역할로 남아 있다. 문제는 현재 많은 버티컬 AI가 이러한 전문가 개입을 일회성 외주 검수나 데이터 구축 비용으로만 처리하여, 누가 어떤 방식으로 품질을 높였는지, 어떤 수정이 이후에도 재사용되는 지식 자산이 되었는지를 체계적으로 남기지 못한다는 점이다. 본 기술은 다중 전문가의 답변과 상호 검수 과정을 구조화해 고품질 감독 신호를 만들고, 각 개입을 기여 원장에 기록하여 품질, 난도, 재사용성, 오류 예방 효과를 반영한 크레딧으로 정산하는 것을 목표로 한다. 이를 통해 일회성 품질 보정을 넘어, 서비스 운영 과정에서 전문가 지식이 지속적으로 축적·재사용되며 AI 품질이 계속 고도화되는 구조를 구현하고자 한다. 세무·회계는 증빙 문서, 규칙 기반 판단, 예외 처리, 최종 책임 구간이 뚜렷해 본 기술의 첫 실증 분야로 적합하다. |
| :---- | :---- |
| **2.2 기술구조** | 메인 기능 1개, 서브 기능 2개, 부가 모듈로 계층화한다: **【메인】 Expert Consensus Pipeline — 합의 기반 학습 데이터 생성 엔진** •   	Expert Answer Pool: 세무 케이스 1건에 전문가 N명이 독립적으로 초안 작성. Upstage Document Parse·Information Extract로 PDF·스캔 세무 문서를 정형 데이터로 변환 후 입력 •   	Peer Review Graph: 전문가가 서로의 답변을 순환 검수하고 수정 diff를 방향 그래프로 기록. 케이스 1건당 N-1회 감독 신호 생성 •   	Consensus Engine: Upstage Solar Pro API를 활용한 합의 정답 후보 선별, 불일치 구간 탐지, 고위험 케이스 분리. 자동 판정 → 관리자 검토 → 수정 이력 감사 로그 3단계 구조  **【서브1】 Knowledge Asset Extractor — 수정 이력 기반 재사용 규칙 추출** •   	수정 로그에서 템플릿화 가능한 패턴·예외 케이스를 우선 추출하여 규칙 DB 구축 •   	추출된 규칙의 후속 케이스 적용률을 측정하여 자산 가치 검증 **【서브2】 Contribution Ledger \+ Credit Scoring — 전문가 기여 추적 및 크레딧 정산** •   	기여 유형(초안 작성 / 검수 / 규칙 생성 / 오류 예방)별 Append-Only 로그 기록 •   	크레딧 산정: 초기 휴리스틱 기반 → 파일럿 데이터 백테스트 보정 → 관리자 최종 검토. 이의제기 절차 내장  **【보안·개인정보 처리】** •   	세무 문서 내 민감정보(주민등록번호, 사업자번호, 납세 내역 등)는 파이프라인 진입 전 비식별화(마스킹) 처리 •   	전문가 접근 권한은 역할 기반 접근 통제(RBAC)로 관리, 세션 및 접근 이력 로그 보관 •   	Contribution Ledger 감사 로그는 SHA-256 해시 체인으로 위변조 방지 및 무결성 보장 **【구조도】**세무 문서 입력 → \[비식별화\] → Document Parse → Expert Answer Pool → Peer Review Graph → Consensus Engine → \[Knowledge Asset 추출 / Contribution Ledger\] → 합의 데이터셋 \+ 크레딧 정산 |
| **2.3 주요기능** | •   	세무 비정형 문서(PDF·스캔) → 정형 데이터 변환 파이프라인 (Upstage Document Parse) •   	케이스 1건 × 전문가 N명 독립 초안 생성 및 순환 검수 자동 배정 •   	불일치 구간 자동 탐지 및 고위험 케이스 분리 (Solar Pro API, 자동 판정 \+ 관리자 검토) •   	합의 정답 도출 및 수정 이력 기반 재사용 규칙 추출 (템플릿화 가능한 패턴 우선) •   	전문가 기여 유형별 크레딧 산정 및 이의제기 절차 포함 정산 대시보드 •   	민감정보 비식별화 및 역할 기반 접근 통제 |
| **2.4 결과물 형상** | ① 세무 합의 데이터셋: 전문가 10인 × 100건, Peer Review Graph 포함 학습용 데이터셋 ② 전문가 검수·정산 대시보드: 기여 유형별 크레딧 시각화, 관리자 검토 및 이의제기 인터페이스 ③ API 기반 합의 엔진: Expert Answer Pool → Peer Review Graph → Consensus Engine REST API |
| **2.5 배포방식** | ① 핵심 엔진: Python 패키지 및 REST API (Upstage Solar Pro API 기반 추론, 별도 파인튜닝 미포함) ② 대시보드: 웹 애플리케이션(SaaS) 형태 ③ 영리 목적 사용 시 라이선스 계약 체결 |
| **2.6 혁신적 요소** | ① 전문가 검수 과정 자체를 학습 데이터로 전환 — 수정 diff를 방향 그래프로 구조화해 데이터 생산성과 품질을 동시에 향상 (케이스 1건 × N명 답변 × N-1회 검수 \= 기존 1:1 대비 수십 배 밀도 높은 감독 신호 생성) ② 전문가 기여를 유형·품질·재사용도 기반으로 정량화하는 크레딧 정산 엔진 — 데이터 생산과 보상 체계를 단일 파이프라인으로 통합한 구조는 기존 어노테이션 플랫폼에 없는 차별점 ③ 합의 엔진의 3단계 구조(자동 판정 \+ 관리자 검토 \+ 감사 로그)로 크레딧 산정 공정성 및 감사 추적 가능성 확보 |
| **2.7 도전적 요소** | ① 문제-정답 1:1 감독학습이 아니라 다중 전문가 답변과 상호 검수 관계를 supervision graph로 기록하는 구조 ② 개별 수정 로그에서 재사용 가능한 규칙·체크리스트·예외 패턴을 추출해 AI가 다시 활용할 수 있는 지식 자산으로 전환 ③ 거래 기여와 지식 자산 기여를 분리하고, 서비스 운영 레이어와 분리된 기여 원장을 통해 설명 가능한 크레딧 정산을 구현 |

**4\. 파급효과**

| 4.1 기술적 파급효과 | ① 단일 정답 감독학습의 구조적 한계를 극복하는 '전문가 합의 기반 Supervision' 방법론 제시 — 파일럿 결과에 따라 학술 논문 출판 및 데이터 수집·정제 관련 특허 출원 검토 예정 ② 전문가 검수 과정의 방향 그래프 구조화 방법론은 AI 개발 내 인간 기여 평가 체계 연구 분야에 기여할 수 있으며, 버티컬 AI 학습 데이터 품질 향상의 새로운 접근 방식으로 확장 가능성을 검토할 예정 ③ 크레딧 산정 알고리즘(기여 유형·품질·재사용도 기반)은 AI 라벨링 경제 내 공정 보상 체계 연구에 적용 가능 |
| :---- | :---- |
| **4.2 사회·산업적 파급효과** | ① 본 기술은 AI로 인해 반복적 전문업무가 줄어드는 산업개편 과정에서, 전문가를 단순 대체 대상이 아니라 **검수자, 상호 검토자, 예외 판단자, 규칙 설계자**로 전환시키는 기술적 기반을 제공한다. 이는 중소형·프리랜서 전문가가 미래 역할로 자연스럽게 이동할 수 있도록 돕는 전환 안전망 구축에 기여할 수 있다. ② 전문가의 직접 처리량뿐 아니라 규칙 생성, 오류 예방, 재사용 가능한 지식 기여까지 기록·정산함으로써, 향후 직접 수행 업무가 줄어들더라도 그들의 전문성이 시스템 안에 남아 일정 기간 인정받는 새로운 참여·보상 모델을 제시한다. 이를 통해 전문가 풀의 지속 가능성과 고신뢰 AI 서비스의 품질 유지에 기여할 수 있다. ③ 세무·회계 분야 파일럿에서는 서류 반려율, 검토 시간, 공제 판단 정확도 등의 Before/After 효과를 측정하고, 결과에 따라 법률, 보험심사, 노무 등 다른 고신뢰 전문영역으로의 확장 가능성을 평가할 수 있다. 또한 기여 정산과 감사 로그를 갖춘 구조는 향후 AI 시대 전문가 보상과 역할 재설계에 관한 정책·제도 논의의 참고 사례가 될 수 있다.  |
