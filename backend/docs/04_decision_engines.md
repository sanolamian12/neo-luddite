# 04. 판단 엔진 (Decision Engines)

케이스 데이터에서 도출한 판단 로직을 코드로 구현한 프로토타입. 두 엔진 모두 동일 패턴:
**`입력 사실관계(dataclass) → 규칙 함수 → 판정 + 근거 + 필요증빙/리스크`**

## 공통 설계 원칙 (Goal 3에서 도출)

1. **기본값 = 부인**: 필요경비·감면·면세는 납세자 입증사항 → 정보·증빙 없으면 부인
2. **3단계 판정**: 전부인정 / 안분인정 / 부인 (+ 조건부)
3. **근거 주석**: 각 판정에 케이스번호·조문 인용 (감사 추적성)
4. **특례 우선**: 업무용승용차·접대비·복리후생 등 별도 한도·규정 먼저 적용

## A. 크리에이터 세무 엔진 — `creator_tax_engine.py`

신고 전 4대 판단 + 1 자동화.

| # | 판단 | 핵심 결정변수 | 케이스 근거 |
|---|---|---|---|
| 1 | 업종코드 (940306 면세 vs 921505 과세) | 근로자 고용·물적시설 보유 | 조심2026서0055 |
| 2 | 부가세 (면세/과세10%/영세율0%) | 매출원천(해외광고·국내광고·후원·굿즈) | 부가세법 §22/§26 |
| 3 | 창업중소기업 감면 (조특법§6) | 인적용역 여부·종전사업이력·연령·지역 | 조심2026중1104, 2026서0581 |
| 4 | 소득종류 (사업 vs 기타) | 계속·반복성 | 소득세법 §19/§21 |
| 5 | 현금매출명세서 (2026.4~) | 현금성 후원 매출 유무 | 신종업종 제도 |

```python
from creator_tax_engine import CreatorFacts, evaluate
d = evaluate(CreatorFacts(has_employees=False, has_physical_facility=False,
                          is_continuous=True, rev_overseas_ad=50_000_000, age=28))
# → 업종코드 940306(면세), 창업감면 배제, 사업소득
```

**핵심 인사이트**: 1인 인적용역(940306)은 창업감면 불가 + 영세율 무의미 →
직원·설비 갖춘 921505로 전환 시 감면·영세율 환급 유불리 비교가 컨설팅 포인트.

## B. 병의원 비용처리 엔진 — `clinic_expense_engine.py`

"이 지출 비용처리 되나?"를 10개 항목 유형으로 판별.

### 입력 스키마 (4종 입력군 + 공통 전제)

```
공통 전제 (ClinicProfile): 사업자유형(개인/법인) · 복식부기 · 성실신고대상
개별 지출 (ExpenseInput):
  1. 명의·계약    in_business_name
  2. 사용 실태     etype + 항목별 swing factor
  3. 안분 측정값   business_use_ratio (운행기록·면적·일정·회선)
  4. 증빙          has_qualified_receipt
```

### 10개 질문 → 항목 유형 매핑

| 질문 | etype | 결정변수 |
|---|---|---|
| Q1 리스차량(가족사용) / Q7 자가차량 | 업무용승용차 | 운행기록부·업무전용보험·승용차특례 |
| Q2 오피스텔 휴게공간 | 임차료 | 실사용자·용도·업무사용비율 |
| Q3 골프 | 접대성지출 | 거래처 동반·상대방 기록·접대비한도 |
| Q4 명품 고객제공 | 광고선전비 | 불특정다수·인당금액(3만)·의료법 유인 |
| Q5 휴대폰 2대 | 통신비 | 회선별 명의·업무사용비율 |
| Q6 헬스장 | 복리후생비 | 전직원 수혜·사규근거 |
| Q8 해외출장(관광) | 출장비 | 공식일정증빙·업무일비율·동반가족 |
| Q9 AI 구독료 | 소프트웨어구독 | 업무사용비율 |
| Q10 자택 사무공간 | 가사관련비 | 업무면적비율·별도사업장 |

```python
from clinic_expense_engine import ClinicProfile, ExpenseInput, ExpenseType, evaluate
r = evaluate(ClinicProfile(성실신고확인대상=True),
             ExpenseInput(ExpenseType.업무용승용차, 18_000_000,
                          운행기록부=True, 업무전용보험=True, business_use_ratio=0.7))
# → 안분인정 1,260만원, 리스크 50/100, 필요증빙 리스트
```

**출력**: 판정(전부/안분/부인/조건부) · 인정금액 · 리스크점수(0~100) · 필요증빙 체크리스트

## 확장 가이드 (프로토타입 빌드 시)

- **새 직업군 추가**: `*_engine.py`를 복제, `dataclass` 입력 스키마 + 규칙 함수만 교체
- **케이스 근거 보강**: `data/processed/all_cases.jsonl`에서 키워드 검색 → 판단기준 추출
  → 규칙 함수의 임계값·근거 주석 업데이트
- **웹/챗 프로토타입**: 엔진의 `evaluate()`를 API로 노출, 입력 스키마를 폼/질문으로 매핑
  (입력 스키마 = 사용자에게 물어볼 질문 목록과 1:1 대응)
- **주의**: 한도 계산(접대비·업무용승용차 1,500만원 등)은 현재 단순화됨 → 실제 한도식·
  연도별 개정 반영 필요. 의료법 환자유인 등 세법 외 규제는 별도 검토.
