# Prototype 3 — 개요

**한 줄 정의:** 챗 사용자와 감사 평가자를 **별도 계정**으로 분리하고, 감사 모드를
**3-pane 어노테이션 워크스페이스**로 재설계하며, 그 안에 **챗 로그 감사**와 **지식 베이스(KB)** 두
주요 표면(surface)을 둔다.

## 액터 (계정)
| 계정 | 역할 | 진입 | 동작 |
|---|---|---|---|
| `viewer` (소상공인) | 챗 사용자 — 프토 1과 동일. **occupation 가변** (직업군 변경 가능). | `/chat/<occupation>` | 결정적 재생·스타터·컴포저 |
| `auditor` (세무 전문가) | 감사 평가자 — 사람. **occupation 무관** (모든 직업군 다룸). | `/audit/...` | 챗 로그 감사 + KB 작성 |

> 두 계정은 **사이드바 푸터**에서 전환한다. 헤더의 "감사 모드 / 챗 모드" 링크 토글은 **제거**한다.
> 프로토타입은 viewer/auditor 각 **1계정**만 둔다(다계정은 범위 밖). viewer의 occupation은 기존 `/select` 플로우로 변경.

## 아키텍처 한눈에
```
┌──────────────────────────────────────────────────────────────┐
│  AppShell (계정 = viewer)                AuditShell (auditor) │
│  ┌─ Sidebar ────────┐                  ┌─ Sidebar ─────────┐ │
│  │ 새 상담          │                  │ 챗 감사            │ │
│  │ 상담 세션 목록    │                  │ 지식 베이스        │ │
│  │ ...              │                  │ ...               │ │
│  │ ─────────────    │                  │ ─────────────     │ │
│  │ [Account] ▾  ←──┼── 푸터 전환기 ────┼── [Account] ▾    │ │
│  └──────────────────┘                  └───────────────────┘ │
│  Main: chat replay                     Main: 3-pane           │
└──────────────────────────────────────────────────────────────┘
```

감사 모드의 본문은 라우트에 따라 두 가지 형태를 가진다:
- `/audit/chat-logs/[conversationId]` — **3-pane** (큐 스트립 · 전사 · 인스펙터)
- `/audit/knowledge/[slug]` — **2-pane** (목록 · 리더/에디터)

## 프토 1·2와의 관계
- **재사용**: 라인 피드백·세션 평가 스키마/스토어/패널 컴포넌트 ([prototype2](../prototype2/README.md)의 A0–A4 산출물).
- **재배치**: 감사 사이드바를 IA 내비로 교체, `audit-experience.tsx`는 3-pane 컨테이너로 재작성.
- **신규**: 계정 모델, KB 스키마/스토어/UI, 시드 로더.
- **삭제**: `FlipToAuditButton`, 헤더의 챗⇄감사 링크.

## 결정 로그

> 본 문서 작성 시점의 기본값. 구현 진행 중 바뀌면 본 문서를 업데이트한다.

| # | 주제 | 결정 |
|---|---|---|
| 1 | 계정 모델 | 두 개 시드 계정 — `viewer` (occupation 가변), `auditor` (occupation 무관, 세무 전문가). 각 1계정만. |
| 2 | 모드 영속 | `activeAccountId`를 localStorage에 영속. 첫 방문은 `viewer`. |
| 3 | `reviewerName` 위치 | `audit-store`에서 `auditor` 계정 객체로 이관. |
| 4 | viewer 직업군 | `viewer.occupation` 필드(가변). `/select` 플로우가 이 값을 갱신. |
| 5 | 감사 상위 내비 | `/audit/chat-logs`, `/audit/knowledge` — 사이드바 헤더 영역의 탭형. `/audit`는 chat-logs로 리다이렉트. |
| 6 | 챗 감사 레이아웃 | **3-pane** — 큐 스트립(좌) · 전사(중) · 인스펙터(우). |
| 7 | **KB 구조** | **Skill 형식** — 마스터 문서(`스킬.md`) + 폴더 기반 보조 문서 트리. 파일명은 한국어. |
| 8 | KB 카테고리 (v1) | `해석론/`, `직업군/<occupation>/`, `판례노트/`, `용어집/`, `오류패턴/` (빈 폴더 슬롯). |
| 9 | KB 스키마 | 경로 기반: `{ path, category, frontmatter{title,summary,tags}, body(md), source, status }`. 본문에서 `[[경로]]` 위키 링크 지원. |
| 10 | 시드 정책 | `docs/02,04*.md` + `canon_examples.json` + 대화 토픽 → 폴더별 시드 매핑(아래 03 문서 참조). 시드는 읽기 전용 — "확장"으로 사본 생성. |
| 11 | 저장 | localStorage Zustand persist (`account-store-v1`, `kb-store-v1`). 백엔드 없음. |
| 12 | 에디터 (v1) | Textarea + 라이브 마크다운 프리뷰 (`react-markdown`). 툴바·자동완성 **없음**. 장기 비전은 "지식 익스플로러"로 별도 트랙. |
| 13 | 라인 피드백 → KB 링크 | 스키마에 `relatedKbIds: string[]`을 **B5**에서 추가 (지금 설계만 반영). |
| 14 | KB → 케이스 링크 | 메타 기반 케이스 피커는 **B6 (선택)**. 현재는 자유 텍스트 인용. |
| 15 | 단계 순서 | B0(계정·내비) → B1(IA) → B2(3-pane) → B3(KB 읽기) → B4(KB 쓰기) → B5(교차링크) → B6(케이스 피커). |

## 범위 밖
- 자동 채점·LLM 기반 보조 (사람 평가만)
- 다중 평가자 동시 편집
- 사용자 인증/권한
- 챗 응답 파이프라인이 KB를 참조 (역방향 — 향후 별도 작업)
- KB 풀텍스트 검색 (간단 키워드 필터까지만)
