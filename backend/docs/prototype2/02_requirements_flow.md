# Prototype 2 — 요구사항 & 사용자 플로우

**한 줄 정의:** 평가자(공급자, 사람)가 유저-AI 대화를 열람하고, **문장 단위로 라인 피드백**을 남기며,
**세션 전체를 정성·정량 평가**하는 인터페이스.

## 액터
- **평가자(공급자)** — 사람. 세무 전문가 등. 유일한 작성 주체.
- (유저-AI 대화는 읽기 전용 입력. 프로토타입 1의 3개 대화/36 세그먼트.)

## 핵심 플로우

```
0. 대화 열람        평가자가 유저-AI 대화 전문을 라인별로 본다
        │
1. 문장 선택        문제 있는 문장을 선택(문장=세그먼트 단위 하이라이트)
        │
2. 피드백 작성      선택 문장에 대한 피드백(코멘트)을 남긴다
        │
3. 태그 부여(선택)  법적 해석 오류 / 문법적 오류 / 제안사항
        │
4. 세션 평가        대화 전체에 대한 평가
   ├ 4.1 정성: 줄글(자유 서술)
   └ 4.2 정량: 카테고리별 점수 — 문장력 · 법률적 정확성
```

- **1~3은 반복**(한 세션에서 여러 문장에 피드백). **4는 세션당 1회** 마무리.
- 선택 단위 = **세그먼트(문장)**. `data-segment-id`로 앵커링(프로토타입 1에서 확보).
  - v1: 한 피드백 = 한 문장. (다중 문장 선택은 후순위)

## 데이터 모델 (Tier 3 — 사람 작성)

```ts
// 라인 피드백 태그 (3종, 다중 선택 가능)
type FeedbackTag = "legal_error" | "grammar_error" | "suggestion";
//  legal_error=법적 해석 오류 · grammar_error=문법적 오류 · suggestion=제안사항

interface LineFeedback {
  id: string;
  conversationId: string;
  segmentId: string;          // ← 문장(세그먼트) 앵커
  reviewer: string;           // 평가자
  body: string;               // 피드백 내용 (필수)
  tags: FeedbackTag[];        // 선택(0개 이상)
  createdAt: string;
}

interface SessionEvaluation {
  id: string;
  conversationId: string;
  reviewer: string;
  qualitative: string;        // 4.1 정성 평가(줄글)
  scores: {                   // 4.2 정량 평가(카테고리별)
    writing: number;          // 문장력
    legalAccuracy: number;    // 법률적 정확성
  };
  createdAt: string;
}
```

- 대화 데이터는 **불변**. 위 두 컬렉션만 신규 생성. 외래키 = `conversationId` / `segmentId`.
- 저장: 프로토타입이므로 **로컬 영속화**(localStorage 또는 JSON export) 권장.

## 화면 구조 (초안)

```
┌───────────────────────────────┬───────────────────────────┐
│  대화 전문 (라인별)           │  피드백 패널                │
│  ─ user 말풍선                │  [선택된 문장]              │
│  ─ assistant 문장들           │   - 피드백 입력(textarea)   │
│    · 문장 hover→하이라이트     │   - 태그 칩(3종 토글)       │
│    · 피드백 있으면 마커/카운트 │   - 저장                    │
│                               │  ─────────────────────────  │
│                               │  세션 평가 (하단/탭)        │
│                               │   - 정성: 줄글              │
│                               │   - 정량: 문장력 / 법률정확  │
│                               │   - 제출                    │
└───────────────────────────────┴───────────────────────────┘
```

- 좌측 = 검토 대상(읽기), 우측 = 작성(피드백+세션평가).
- 문장에 피드백이 달리면 좌측에 시각 표시(밑줄/배경 + 개수).

## 확정된 결정사항
1. ✅ **정량 점수 척도**: **1–5**.
2. ✅ **정량 카테고리**: **문장력 · 법률적 정확성** 2종으로 시작(확장 여지).
3. ✅ **태그**: **3종 고정** (법적 해석 오류 · 문법적 오류 · 제안사항).
4. ✅ **근거 패널(Tier 2)**: **후순위** (핵심 플로우 먼저).
5. ✅ **다중 문장 선택**: v1 **단일 문장**. 다중 선택은 **별도 기능(후속)**.
6. ✅ **진입**: 사이드바 세션 목록에서 대화 선택 → 검토(Audit) 모드 진입.

## 모드 분리 — Chat 모드 vs Audit 모드 (★ 핵심 아키텍처)

**챗 사용자와 감사 평가자는 서로 다른 사용자다.** 따라서 같은 대화를 두 모드로 본다:

| | Chat 모드 (프로토타입 1) | Audit 모드 (프로토타입 2) |
|---|---|---|
| 사용자 | 최종 유저 (소상공인) | 평가자(공급자) |
| 동작 | 결정적 재생·스타터·컴포저 | 전문(全文) 표시·문장 선택·피드백·세션 평가 |
| 입력 | 새 질문 | 기존 대화 검토 |

- **모드 전환 UI("flip")**: 대화 화면에서 Chat ⟷ Audit 모드를 전환하는 컨트롤.
- 같은 대화 데이터(불변)를 공유하되, **렌더·인터랙션 레이어만 교체**.
- 권장 구현: 별도 라우트(예: `/audit/[conversationId]`) + 헤더의 모드 토글.
  - 실제 제품에서는 두 모드가 **별도 사용자 계정/권한**이지만, 프로토타입에선 토글로 시연.
- Audit 모드에서 전문이 **즉시 전부 공개**(재생 애니메이션 없이) → `revealAll` 활용.

## 범위 밖 (Non-goals, v1)
- 시스템 자동 채점/판정 (평가는 사람만).
- 다중 문장 선택 (별도 후속 기능).
- 실시간 협업·멀티 평가자 동시 편집.
- 서버 DB(백엔드) — 로컬 영속화로 충분.
- 사용자 인증/권한 분리 (실제 제품 영역; 프로토타입은 모드 토글로 대체).
