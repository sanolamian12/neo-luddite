# Prototype 2 — 챗 라인 단위 검토 & 피드백 (사람 평가자)

**사람 평가자**가 챗 대화를 **문장(세그먼트) 단위로 검토**하고 **피드백/코멘트**를 다는 인터페이스.
시스템 자동 판정이 아니라 **사람의 판단**을 기록하는 것이 핵심. 프로토타입 1의 대화 데이터(`data-segment-id`)를
그대로 입력으로 사용한다.

## 전제 (프로토타입 1에서 확보됨)
- 모든 assistant/user 문장이 `data-segment-id`로 DOM에 노출 → 라인 앵커링 준비 완료.
- 대화 데이터는 불변 JSON, 피드백은 `segmentId` 외래키 별도 컬렉션으로 분리(권장).
- 인용 케이스는 1,301건 코퍼스에 존재 → 평가자가 **근거 패널에서 직접 확인** 가능(자동 판정 아님).
